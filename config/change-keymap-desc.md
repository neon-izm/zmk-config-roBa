# change-keymap-desc

## 概要

現在の `roBa.keymap` は、次の前提で調整されています。

- Windows の日本語キーボード配列をそのまま使う
- プログラミング用途を優先する
- roBa のロータリーエンコーダは使わない
- トラックボールはマウス操作、スクロール、ズームに使う

そのため、頻出する英字や記号はベースレイヤーと記号レイヤーで安定して打てるようにし、トラックボール系の操作は親指キーの Hold で呼び出す構成になっています。

## レイヤー構成

### 1. default_layer

ベースレイヤーは、英字入力と日常的な編集操作を優先した構成です。

- `A`, `P`, `;`, `/` は単独タップで入力できる
- 右 Shift は廃止し、`/` は単独キーになっている
- `[` と `]` は右手内側に配置
- 最下段は `Ctrl`, `Win`, `Alt`, 無変換, Space, 変換, Backspace, Enter, Delete` の並び

親指キーの役割は次の通りです。

- 無変換タップ: `INT_MUHENKAN`
- 無変換ホールド: `SCROLL` レイヤー
- Space ホールド: `NUM` レイヤー
- 変換タップ: `INT_HENKAN`
- 変換ホールド: `Ctrl + SCROLL` によるズーム操作
- Enter ホールド: `ARROW` レイヤー
- Delete ホールド: `FUNCTION` レイヤー

### 2. FUNCTION レイヤー

FUNCTION レイヤーはファンクションキー専用です。

- 上段に `F1` から `F10`
- 右端に `F11`, `F12`, `F13`
- 左手側に `ESCAPE`

### 3. NUM レイヤー

NUM レイヤーは、数字とプログラミングでよく使う記号をまとめたレイヤーです。

- 数字 `0` から `9`
- `_`, `-`, `=`, `+`
- `` ` ``, `~`, `^`, `&`, `@`, `:`
- `(`, `)`, `[`, `]`, `{`, `}`, `|`
- `'`, `"`, `¥`

日本語配列を前提にした `JP_*` マクロを使うことで、物理配列と入力される記号のずれを吸収しています。

### 4. ARROW レイヤー

ARROW レイヤーは、コード編集向けの移動操作をまとめたレイヤーです。

- 矢印キー
- `Ctrl+Tab`, `Ctrl+Shift+Tab`
- `Home`, `End`, `Ctrl+Home`, `Ctrl+End`
- `Ctrl+Left`, `Ctrl+Right`
- `Page Up`, `Page Down`
- `Delete`, `Backspace`
- `Shift+Home`, `Shift+End`

単純なカーソル移動だけでなく、タブ切り替え、単語移動、行頭行末移動まで一つのレイヤーで扱えるようにしています。

### 5. MOUSE レイヤー

MOUSE レイヤーはトラックボール前提のマウス操作用レイヤーです。

- 右手側に `MB1`, `MB2`, `MB3`
- 右手側に `Shift`, `Ctrl`, `Alt`
- 左手側のキーは `&to 0` で `default_layer` に戻る

この構成により、トラックボールを触って自動で MOUSE レイヤーに入っても、左手のどれかのキーに触れればすぐ通常入力へ戻れます。

### 6. SCROLL レイヤー

SCROLL レイヤー自体には個別キーをほぼ置かず、トラックボールのスクロール動作を有効化するためのレイヤーとして使っています。

- 無変換キーを Hold している間だけ有効
- `hold-while-undecided` を設定しているため、押した直後からスクロールを認識しやすい

### 7. ZOOM 操作

ズームは専用レイヤーではなく、変換キー Hold で呼び出す `Ctrl + SCROLL` マクロとして実装しています。

- 変換キーを Hold すると `Ctrl` を押しながら `SCROLL` レイヤーを有効化
- その状態でトラックボールをスクロールすると拡大・縮小になる想定

## コンボ

現状の主なコンボは次の通りです。

- `Tab`
- `Shift+Tab`
- `-`
- `_`
- `layer_6` への移動
- `Escape`
- `Undo`
- `Redo`

編集中によく使う操作をホームポジション付近から呼び出せるようにしています。

## 補足

- ロータリーエンコーダは物理的に使わない前提のため、このキーマップでは主要操作に使っていません
- トラックボール関連の操作は、エンコーダではなく親指 Hold と自動マウスレイヤー遷移を中心に組んでいます
- `keymap-drawer/roBa.yaml` はこの現行構成に合わせて更新済みです

## まとめ

現在の構成は、JP配列 Windows 環境でのプログラミングを優先して次の方針でまとめています。

- ベースレイヤーでは文字入力を阻害しない
- 記号は NUM レイヤーに集約して覚えやすくする
- 編集移動は ARROW レイヤーで完結させる
- トラックボール操作は親指 Hold でスクロールとズームを呼び出す
- マウスレイヤーからは左手ですぐ通常入力へ戻れるようにする
